# MECE Principle

Mutually Exclusive, Collectively Exhaustive. Used to break down a large problem into non-overlapping and comprehensive sub-problems.

- Binary: Two parts (e.g., domestic/foreign)
- Process: Time sequence (e.g., SOP)
- Elements: Key components
- Formula: Math (e.g., Total Time = Frontend + Network + API)
- Matrix: 2 dimensions, 4 quadrants (e.g., Important/Urgent)

---

# Attribution Tracing

A method to find the real cause by **step-by-step elimination and analysis** of multiple possibilities, starting from the result.

<br>

**üå∞: A video app's playback stability suddenly dropped by 30%. How to find the issue?**<!-- .element: class="fragment" data-fragment-index="1" -->

<br>

ü§î: System? -> Network? -> Phone brand? -> App version?<!-- .element: class="fragment" data-fragment-index="2" -->

# How to judge if it's clarified?

- Any confusion left?
- Any overlooked points?
- Do you feel completely untroubled?
- Can you answer any argument from others?

# How to write clear documentation

1. **Logical structure**: Headings, subheadings, mind maps first.
2. **Simple expression**: Clear, logical, and easy-to-understand language.
3. **Appropriate examples**: Real or simulated cases, demos, flowcharts.

<br><br>How to judge: Others can start coding just by looking at your doc ü§ì<!-- .element: class="fragment" data-fragment-index="1" -->

---

# Check RFC

Request for Comments, a format for internet standards.
</br></br>

1. Clarify target audience and purpose.
2. Describe background and existing solutions.
3. Elaborate on the solution with logic and details.
4. Note unresolved issues and future possibilities.

# STAR Model for Communication

- Situation: The specific **context**.
- Task: The problem or **task** at hand.
- Action: The **actions** taken.
- Result: The **result** of those actions.

<br><br>
üå∞: How we plan to solve the performance issue of XXX?<!-- .element: class="fragment" data-fragment-index="1" -->

# SCQA Model for Persuasion

- Situation: Start with a familiar **context**.
- Complication: A **conflict** or obstacle.
- Question: What is the **problem**?
- Answer: Our **solution**.
